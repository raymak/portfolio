Contextual Feature Recommender

Contextual feature recommender (CFR) makes users aware of the features of a system that could improve their experience. It observes users' behavior, infers what features they might benefit from, and eventually recommends these features to them in an unobtrusive way at the right moment.

Motivation Any non-trivial system contains various tools that each user only benefits from a unique subset of them. We can use the correlation between users' need of these tools and patterns of their usage to proactively find and recommend the features that are tailored to their very personal experience of the system.

First experiment I devised the idea and developed the first experimental prototypes of this system during my internship at Mozilla in Summer 2014. I implemented multiple variations of CFR on Firefox. It recommended features such as keyboard shortcuts, extensions, private browsing, etc. to the users if the user was likely to benefit based on their behavior. We then conducted an experiment on about 16000 users, collected the data and did a preliminary analysis.

Code The code is open-source and is hosted on github.

Tools The prototypes are implemented in javascript using Firefox addon SDK. We use Mozilla's Test Pilot platform to remotely deploy the addon. I have used Python for data processing.

Status Extending this project and doing further experiments is currently my main research and it is closely related to my Master's thesis topic.

I am now continuing to work on this project as my Master's thesis. This problem is essentially interesting because it involves many of the challenges of designing proactive data-driven intelligent systems such as obtrusiveness, privacy invasion, lack of intelligibility, etc.
